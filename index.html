Why ctx?

1

Single source of truth

Define your AI context once in .ctx/. No more duplicating rules across .claude/, .cursor/, and .windsurf/ directories.

2

Team-friendly

Commit .ctx/ and ctx.yaml to git. Platform output is gitignored. Teammates just run ctx sync after pulling.

3

Multi-platform

Supports Claude Code, Cursor, and Windsurf out of the box. Each platform gets files in its native format.

4

Smart sync

Content hashing tracks what changed. Only modified slots are re-synced. A lockfile keeps everything in check.

5

Four slot types

Organize context into agents, rules, memory, and docs — each mapped to the right platform location.

6

Diff before sync

Preview exactly what will change with ctx diff before committing to a sync. No surprises.

Commands

ctx init Initialize a .ctx/ folder and ctx.yaml manifest in your project.
ctx add Add a context slot — choose from agent, rules, memory, or docs.
ctx sync Compile and distribute context to all configured platforms.
ctx status Show sync status of every context slot across platforms.
ctx diff Preview what would change on the next sync before running it.

Get Started in 60 Seconds

1

Initialize

Run ctx init --platforms claude-code,cursor,windsurf in your project root.

2

Add context slots

Use ctx add rules testing or ctx add agent reviewer to create context files.

3

Write your context

Edit the markdown files in .ctx/ with your team's conventions, agent prompts, and docs.

4

Sync

Run ctx sync to generate platform-specific files. Commit .ctx/ and share with your team.
